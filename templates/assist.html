Assist - Understanding the Intrusion Detection System

Importance of This Website

This Intrusion Detection System (IDS) helps to detect malicious activities in a network by analyzing traffic data and identifying threats. With machine learning, it improves accuracy in real-time threat detection, reducing manual monitoring and enhancing cybersecurity.

Feature (Column) Descriptions from KDD Dataset

  • duration: length (in seconds) of the connection
  • protocol_type: type of protocol (e.g., TCP, UDP)
  • service: network service used (e.g., HTTP, FTP)
  • flag: status of the connection
  • src_bytes: bytes sent from source to destination
  • dst_bytes: bytes sent from destination to source
  • count: number of connections to the same host in past 2 seconds
  • srv_count: number of connections to the same service in past 2 seconds

Common Attack Types

  • neptune: DoS attack using SYN flooding
  • smurf: ICMP-based broadcast flooding (DoS)
  • satan: Probing for vulnerabilities in services
  • ipsweep: Scanning multiple IPs to find active hosts
  • nmap: Port scanning to discover open services
  • portsweep: Scanning multiple ports on a single host
  • back: DoS attack using backdoor communication
  • teardrop: Malformed packet fragmentation (DoS)
  • warezclient: Unauthorized access to upload/download files
  • guess_passwd: Repeated login attempts to guess passwords
